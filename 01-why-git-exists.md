# Why Git exists

## Overview

Git is a version control system.

It is used to track changes in files, save project history, and manage different versions of a project.

Without Git, people often manage files manually using copied folders or repeated filenames. This creates confusion and makes it difficult to know which version is correct.

## Problem without Git

Manual file management often creates problems such as:

* Multiple duplicate files
* Confusing file names
* No clear project history
* Difficulty recovering old versions
* Difficulty tracking who changed what
* Difficulty working with others

Example:

```text
project-final.docx
project-final-v2.docx
project-final-updated.docx
project-final-latest.docx
project-final-latest-new.docx
```

This method is not reliable for technical work.

## What Git solves

Git solves these problems by tracking changes properly.

Git allows users to:

* Save project progress using commits
* View previous versions
* Compare changes
* Restore older versions
* Work on separate branches
* Share work through remote platforms like GitHub
* Collaborate with other people

## Key concepts

| Term              | Description                                               |
| ----------------- | --------------------------------------------------------- |
| Version control   | A system for tracking file changes over time              |
| Repository        | A folder tracked by Git                                   |
| Commit            | A saved snapshot of project changes                       |
| History           | The record of commits in a repository                     |
| Branch            | A separate line of development                            |
| Remote repository | A copy of the repository stored online, such as on GitHub |

## Git and GitHub

Git and GitHub are not the same.

| Tool     | Purpose                         |
| -------- | ------------------------------- |
| `Git`    | Tracks changes in files locally |
| `GitHub` | Hosts Git repositories online   |

Git works on the computer.

GitHub stores and shares repositories online.

## Practical example

A system administrator may use Git to track configuration files.

Example files:

```text
nginx.conf
backup-script.sh
server-notes.md
network-checklist.md
```

Git helps track when these files were changed and why the changes were made.

## Why this matters for IT roles

Git is useful in IT Support, System Administration, and DevOps because technical work often involves:

* Scripts
* Configuration files
* Documentation
* Troubleshooting notes
* Automation files
* Project records

Using Git makes these files easier to manage and review.

## Notes

* Git tracks changes, not just files.
* Git keeps a history of work.
* Git helps avoid confusion from duplicate file versions.
* GitHub is used to store and share Git repositories online.
* Good commit messages make project history easier to understand.
